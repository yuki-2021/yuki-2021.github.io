ViewResolver和View

🍎 View解析和@ResponseBoady区别

1️⃣ 设置**响应的Content-type响应头。\**显示指定就下一步\**，没有显示指定，就根据req的Accept自动设定**。

2️⃣ 有@ResponseBody,直接**使用Converter转换**，然后返回。

3️⃣ 没有，使用**ViewResolver找到合适的View，然后渲染内容**，返回。

🍟 传递数据给View - Model、**ModelMap**、**Map**、**Pojo**

Model、**ModelMap**、**Map**、**Pojo** 会自动传递给View。

  • Map、ModelMap、Model作为参数无区别，spring-mvc在运行时创建的都是**BindingAwareModelMap**。
  • 其他类型不会传递

🍒 ModelAndView - 指定view和model

🌶️ 请求转发 - redirect:/hello

🍚 重定向和**RedirectAttributes**- redirect:/hello

RedirectAttributes - **传递重定向参数**。

**@ModelAttribute**用于**接收重定向参数**。

🐳 @CookieValue - 获取Cookie

🐋 @SessionAttribute - 获取Session

🌺@DateTimeFormat - 按照parttern解析str到Date
