Spring获取Bean实例

可以在BeanFactory和ApplicationContex中获取，区别如下

  • 初始化时机。BeanFactory获取时初始化，ApplicationContext创建时初始化。
  • ApplicationContext提供了诸如事件传递、国际化、资源加载等功能。

demo1- 获取用户列表

创建Bean的方式

普通bean - id、name、class
  • id和name表示用来标志唯一的Bean
  • 可以同时有id和name
  • name和id不能重复
工厂方法Bean - factoryMethod
简单工厂Bean - factoryBean和factoryMethod
abstrct - 抽象Bean
  • 配置了abstrct的类不能实例化。
  • 一般和parent一起使用，作为通用配置。
parent - 继承配置
depends-on - 指定启动顺序

depends-on来指定实例化顺序。

依赖注入

属性注入 - ref、value、内部bean
  • ref注入引用类型
  • 内部Bean用于注入只使用一次或者需要匿名的bean
  • value注入值
构造注入 - construct

默认情况如下

方法参数的类型不重复，可以使用type。

方法参数类型重复，使用name。

注入null
其他类型 - set/list/array/map/props

Bean的生命周期

Bean初始化时候，会经历如下生命周期

  • STEP1 - 调用构造函数或者factory-method。
  • STEP2 - 设置属性，调用setXXX()。
  • STEP3 - 调用BeanNameAware的setBeanName()
  • STEP4 - - 调用ApplicationContextAware的setApplicationContext()
  • STEP5 - 调用BeanPostProcessor的beforeInit，表示开始init初始化。
  • STEP6 - 调用InitlizingBean的afterPropertySet，表示属性设置完毕。
  • STEP7- 调用init-method
  • STEP8 - 待用BeanPostProcessor的afterInit,表示init完毕。
  • STEP9 - 用户可以使用Bean了
  • STEP10 - 调用DisposableBean的destory方法
  • STEP11 - 调用destory-method方法

Bean作用域scope - prototype/singleton/request/session
singleton 单例
prototype 每次请求创建一个
request 每次请求一个(很少用)
session 每个session创建一个(很少用)

对于Web应用，Dao层、controller层、sevice层的对象基本都是单例的。

如果用户需要会话Session，我们会自己创建Session，把实例放到Session中。
