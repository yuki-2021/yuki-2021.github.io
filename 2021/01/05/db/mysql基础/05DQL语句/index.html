DQL语句是Data Query Language，用于查询数据。

基础查询

基础的查询语法是

AS - 起别名
  • 给运算结果起别名
  • 给列起别名
  • 给表起别名
Distinct - 去重复
  • Distinct 紧跟在Select后面
  • 单列，一列作为整体，去重。多列，多列作为整体，去重。
分组 - group by
  • 一般配合聚合函数(count、max、min、avg、sum)，实现分组后统计
  • 多次使用，表示多个分组条件。比如，统计每个部门的每个工种的平均工资，就是部门和工种
Having - 子查询
  • having用来对group by结果进行过滤。
排序 - order by
  • 可选值有ASC、DESC。ASC可以省略。
  • 多次使用，先按c1排，然后按照c2排。
Limit - 分页
  • 语法是Limt(idx,count)。idx省略表示取前count条。
  • idx从0开始，比如取11-20条，应该是limit(10,10)。

条件表达式有>、<、=、!=、<>

逻辑表达式

逻辑表达式有and、or、not。可以连接两个条件。

查询1： 查询 工资在 1w 到 2w 的 员工名 工资 奖金。

查询2： 查询 部门编号 不再 90 - 100 或 工资 高于 15000 的 信息

模糊查询
  • 模糊查询使用like
  • 可用符号有_和%
  • 转义符号是\

查询1： 查询 员工姓名 包含 a 的数据

查询2： 查询 第二个字符是’_’的员工。

Between ... and 和 in

这两个用于范围查询。

查询4： 查询工种编号是 IT_PROG，AD_VP，AD_PRES的员工姓名 和 工种号

IS NULL 和 全等
  • 判断是不是Null，用Is Null，不能用==。
  • <=>是全等，可以当作等号，还能当作isNull。

按照连接条件又分为等值连接、非等值链接。

SQL92 标准仅仅支持内连接、交叉连接。

SQL99 标准 支持内连接、外连接、交叉连接。
