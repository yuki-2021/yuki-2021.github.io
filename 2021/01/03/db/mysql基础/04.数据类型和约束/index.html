数据类型

Mysql数据类型分为整型、小数、字符型、日期型

整数类型

整数类型分为tinyint、smallint、mediumint、int、bigint

tinyint -2^7 ~ 2^7 -1
smallint -2^15 ~ 2^15 -1
midumint -2^23 ~ 2^23 -1
int -2^31 ~ 2^31 -1
bigint -2^63 ~ 2^63 -1
zerofill

可以配合zerofill进行0填充，在长度不够的时候，前面会补充0。

小数类型

小数类型有float(M,D)、double、decimal。

  • 浮点数，M和D默认没有限制。定点数，默认M=10,D=0。可以理解decimal = float(10,0)
  • 对于float和double指定M和D后，和decimal没有区别。
  • 插入数据，小数超过长度，自动截断。位数不够，自动补充0。
float -3.4E^38 - 3.4E^38
double -1.7E^308 - 1.7E^308
decimal 默认相当于double(10,0)

字符类型

字符类型包括char、varchar、tinyText、tinyBlob、blob、text、mediumtext、mediumBlob、longText、longBlob。

char和varChar

char是定长的，var是不定长度的。

  • 使用char(n)或者varChar(n)指定最多存储字符个数
  • char最大空间是255byte，varChar是65535。
char 最大225byte 定长，效率高
varchar 最大65535byte 不定长度，效率低
其他文本

存储大文本用tintText、text、mediumText、longText，差别只是存储内容多少区别

tinaytext 2^8byte,对应85个utf8字符
text 2^16byte
mediumtext 2^24byte
longtext 2^32byte

分别和tinyblob、blob、mediumblob、longblob对应，用来存储二进制。

set和enum

enum表示枚举。

  • 创建表时，声明值的范围
  • 插入时候，只能插入声明值的一个
  • 查询时候，可以使用enum对应的int值查询。
  • 约束值本来是check的功能，但是mysql不支持。有需求时，可以使用enum或者trigger。

在查询的时候，可以使用enum对应的整数查询

set类型

set类型表示集合类型

  • 创建表，声明值的范围
  • 插入数据，只能插入声明值的一个或多个
  • 查询时候，使用内置函数查询

日期类型

包括year、date、time、timedate、TIMESTAMP。

类型 范围 插入 查询
year 1901-2155 插入超出范围，报错 查询出来
date 0-00-00 到 9999-12-12。错误时间的时间戳是0。 分隔符做符号,比如y@m@d。插入值是str或者date。 查询结果自动格式化成str
time
datetime 0-00-00 到 9999-12-12。
错误时间的时间戳是0。		 分隔符做符号,比如y@m@d。
支持插入值是str或者date。		 查询结果自动格式化成str
timestamp 最大2147483647 默认是当前时间。
支持插入str或者date。		 查询结果自动格式化成str

约束

  • 包括null、default、unique、primaryKey、foreignKey，不支持check。
  • primaryKey和foreignKey一般显示指定constraint名称
null 值不能是null
default 默认值
unisque 值唯一
primaryKey 主键
foreignKey 外键
