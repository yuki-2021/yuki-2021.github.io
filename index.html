vue-router进阶

نُشر في 2021-09-14

钩子函数

导航守卫就是钩子函数。分为三类

  • 全局钩子
  • 组件钩子
  • 路由钩子

全局钩子

全局钩子包括

  • router.beforeEach(to,from,next) - 在导航前被执行，需要手动调用next()。
  • router.afterEach(to,from) - 在导航后被执行

next()支持如下

  • next() - 会正常跳转
  • next(false/true) - 如果是false，不跳转(回到From)。如果是true，正常跳转。
  • next({ path: '/' }) - 跳转到path

组件钩子

组件钩子包括

  • beforeRouteEnter() - 进入组件触发，在created()之前被调用。不能访问this。可以通过next(vm => {xxx})方式传递回调(在created后执行)。
  • beforeRouteLeave() - 组件离开时触发。

beforeRouteEnter()可以用来获取数据。

route钩子

route钩子比较简单，在进入route之前调用。

  • beforeEnter() - 进入route之前调用。

执行顺序

从组件A(/a)进入组件B(/b)，执行顺序如下：

  • A组件的beforeRouteLeave()
  • beforeEach()
  • beforeEnter()
  • afterEach()
  • beforeRouteEnter()
  • created()、mounted()

元数据

元数据就是metadata。

  • 在router中使用meta传递
  • 使用this.$route.meta访问

路由懒加载

路由懒加载，效果是

  • 把组件分离成js
  • 需要的时候，进行加载

实现方式如下

  • 通过() => require(xx)引入
  • 传递给route

vue-router使用

نُشر في 2021-09-14

使用

使用分三步

  • 用Vue.use(VueRouter)引入VueRouter
  • 创建VueRouter实例，提供配置。
  • 提供给Vue实例

VueRouter

VueRouter表示vue-router，包含了如下参数

  • path - 路径，支持:xxx(动态路由)、*(通配符)、?p1=v1&p2=v2(查询参数)
  • name - 可以实现命名路由(按照name进行映射)
  • component - 组件
  • components - 可以实现命名视图(把一个path映射到多个Router-View)
  • children - 可以实现嵌套路由(url映射到组件，组件内部提供router-view,同时可以映射子组件)
  • redirect - 重定向
  • alias - 别名

提供了，如下标签

  • <router-link to='xx'> - 解析成<a>标签
  • <router-view> - 会在router-view中，展示Component

最后，提供了$router和$routes两个对象。

  • $router - 有push()和go()方法，
  • $route - 访问params和queryPrams

path - 路径

path 用于把url映射到component。

  • 支持*通配符，用this.$route.params.pathMatch访问匹配的路径。
  • 支持:xxxx，实现动态路由，用this.$route.params访问参数。
  • 在你使用this.$router.push时候，可以把params和query放到path串中，也可以放在参数中
  • path和param不能一起使用，如果需要，放到path串中。

支持通配符

在this.$router.push中，param和path不能一起用。

path - 动态路由

当path中有:xxx，这时候，就叫动态路由。

  • 在path中，使用:xxx
  • 在组件中，用this.$routes.params访问参数。

如果，你用了this.$route.push({xxx})。

  • path和params不能一起用
  • 都写到path串中

name - 命名路由

可以根据name进行组件映射。如果有name，就叫命名路由。

  • 在routes中，定义name。
  • 使用<router-link to='{name: 'xxx', params: {xxx}, query: {xxx} }'>进行访问
  • 使用this.$router.push({name: 'xxx',params: {xxx}, query:{xxx} })进行访问

用<router-link to='xxx'>导航。

用this.$router.push()访问

components - 命名视图

命名视图，是把url映射到多个组件。

  • 使用<router-view name='xxx'>定义多个
  • 用components来把url映射到多个组件

children - 嵌套路由

嵌套路由，使用children实现。

  • 在routes中定义children
  • 在组件中定义<router-view>
  • 当url映射到组件，在组的<router-view>中映射children

redirect - 重定向

alias - 别名

mode - 路由模式

支持hash和history两种模式

  • hash模式，url中有#，使用url锚点实现。
  • history模式，使用history.pushStateApi()实现，直接访问url，会返回404。

区别如下

  • url不同
  • 当部署后，如果是history模式，访问不存在的url，history会报错。对于hash模式，访问不存在的url，hash模式返回首页(因为#的原因)。

  • 在v-enter-active、v-leave-active写过度状态
  • 写<transition name='xx'/>标签

动画名称

两种方式。

  • 粗粒度 - 使用name
  • 细粒度 - 使用enter-class、enter-active-class、enter-to-class、leave-class、leave-active-class、leave-to-class

时间

可能，你在enter-active或者leave-active中

  • 同时指定了animation和transition
  • 他们的时间不一样
  • 如果不指定时间，会各自执行各自的。

使用<transition type="animation|transition">来指定以哪个时间为准。

或者使用duration指定过度时间。

分别指定enter和leave的时间。

多元素

多元素动画 - 过度模式

当有多个元素，可以指定过度模式，决定哪种先过度、哪种后过度。

  • 默认 - 同时生效。enter-active和leave-active同时进行。
  • in-out - 先执行enter-active，后执行leave-active。
  • out-in - 先执行leave-active，后执行enter-active。

列表动画

对于v-for，如果想使用动画，使用<transition-group />。

  • tag - <transition-group />的最外层，会渲染成tag标签
  • v-move - 可以定义v-move，来设置原来的元素的位置
  • move-class - 如果不想用v-move，使用move-class来指定过度

对于v-for，一定要指定:key，每次是给新的key加动画，给旧的key加v-move。

事件修饰符

在js中，经常用到event.preventDefault()或event.stopPropagation()等事件方法。

在Vue中，可以使用事件修饰符。

  • .prevent - 阻止默认行为
  • .stop - 阻止事件传播(用在内部)。对于冒泡浏览器(chrome)，阻止冒泡。对于捕获浏览器(ie)，阻止捕获。
  • .capture - 用在外部元素，这个元素(及内部)的事件模型，变成捕获模型。
  • .self - 自己生效(用在内部)
  • .once - 生效一次

按键修饰符

监听有两种按键修饰符。

  • 按键码
  • 按键名称

按键码

当使用键盘事件，需要按倒指定键，然后触发事件，否则不触发，可以使用按键码。

常用按键码，有按键码别名。

  • .space、.tab、.enter - 空格、制表、回车
  • .delete (捕获“删除”和“退格”键)
  • .esc
  • .up 、.down、.left、.right
  • 其他的，直接用按键名称

最后，使用Vue.config.keyCodes.x=x来定义不存在的按键别名。

系统修饰键

系统修饰符，对所有事件生效，同时按住按键，才能触发事件。

  • .ctrl
  • .alt
  • .shift
  • .meta - 在 Mac 系统键盘上，meta 对应 command 键 (⌘)。在 Windows 系统键盘 meta 对应 Windows 徽标键 (⊞)

鼠标修饰符

顾明思意，必须按下对的鼠标键，才会触发。

  • .right
