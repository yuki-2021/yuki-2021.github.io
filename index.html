Vue动画

transition

在vue中，如果你要用animation动画。

  • 写v-enter-active、v-leave-active的动画样式
  • 然后，写<transition name='xxx'>的过度标签

如果你用transition动画

  • 在v-enter、v-eneter-to、v-leave、v-leave-to写开始状态和结束状态
  • 在v-enter-active、v-leave-active写过度状态
  • 写<transition name='xx'/>标签

动画名称

两种方式。

  • 粗粒度 - 使用name
  • 细粒度 - 使用enter-class、enter-active-class、enter-to-class、leave-class、leave-active-class、leave-to-class

时间

可能，你在enter-active或者leave-active中

  • 同时指定了animation和transition
  • 他们的时间不一样
  • 如果不指定时间，会各自执行各自的。

使用<transition type="animation|transition">来指定以哪个时间为准。

或者使用duration指定过度时间。

分别指定enter和leave的时间。

多元素

多元素动画 - 过度模式

当有多个元素，可以指定过度模式，决定哪种先过度、哪种后过度。

  • 默认 - 同时生效。enter-active和leave-active同时进行。
  • in-out - 先执行enter-active，后执行leave-active。
  • out-in - 先执行leave-active，后执行enter-active。

列表动画

对于v-for，如果想使用动画，使用<transition-group />。

  • tag - <transition-group />的最外层，会渲染成tag标签
  • v-move - 可以定义v-move，来设置原来的元素的位置
  • move-class - 如果不想用v-move，使用move-class来指定过度

对于v-for，一定要指定:key，每次是给新的key加动画，给旧的key加v-move。

Js动画

Js动画

对于Js动画，你提供了如下

  • 使用:css='false'来禁用css动画
  • before-enter、enter、after-enter，和beore-leave、leave、after-leave的钩子
  • 对于enter和leave中，使用done()后，才会执行下一个钩子

事件修饰符

在js中，经常用到event.preventDefault()或event.stopPropagation()等事件方法。

在Vue中，可以使用事件修饰符。

  • .prevent - 阻止默认行为
  • .stop - 阻止事件传播(用在内部)。对于冒泡浏览器(chrome)，阻止冒泡。对于捕获浏览器(ie)，阻止捕获。
  • .capture - 用在外部元素，这个元素(及内部)的事件模型，变成捕获模型。
  • .self - 自己生效(用在内部)
  • .once - 生效一次

按键修饰符

监听有两种按键修饰符。

  • 按键码
  • 按键名称

按键码

当使用键盘事件，需要按倒指定键，然后触发事件，否则不触发，可以使用按键码。

常用按键码，有按键码别名。

  • .space、.tab、.enter - 空格、制表、回车
  • .delete (捕获“删除”和“退格”键)
  • .esc
  • .up 、.down、.left、.right
  • 其他的，直接用按键名称

最后，使用Vue.config.keyCodes.x=x来定义不存在的按键别名。

系统修饰键

系统修饰符，对所有事件生效，同时按住按键，才能触发事件。

  • .ctrl
  • .alt
  • .shift
  • .meta - 在 Mac 系统键盘上，meta 对应 command 键 (⌘)。在 Windows 系统键盘 meta 对应 Windows 徽标键 (⊞)

鼠标修饰符

顾明思意，必须按下对的鼠标键，才会触发。

  • .left
  • .right
  • .middle

v-show

v-show和v-if区别

v-show的实现方式是，把模板渲染到Dom，如果条件成立，显示。如果条件不成立，设置display:none属性。

v-if则是，如果不显示，就在Dom中剔除。

也就是说，v-show在第一次渲染时，会开销大。但是在切换时，开销小。

元素复用

复用

在上面的例子中，当我把login_type由username切换到phone，Vue算法是如下做的切换

  • 检查旧的v-if块和新的v-if块，筛选出相同的标签(key不同，不是相同的标签)

  • 把<label>中的username换成phone

而不是

  • 删除div
  • 添加新的div

这样可以复用元素，避免Dom开销。

不复用 - key

如果你确实希望不复用v-if块的内容，可以加上key属性，表示他们是不同的标签，就不会复用了。
