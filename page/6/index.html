特点如下
| | NAT | Bridge | Internal | Host Only |
| —————— | —- | —— | ——– | ——— |
| 虚拟机 -> 主机 | √ | √ | × | 需要设置 |
| 主机 -> 虚拟机 | × | √ | × | 需要设置 |
| 虚拟机 -> 其他主机 | √ | √ | × | 需要设置 |
| 其他主机 -> 虚拟机 | × | √ | × | 需要设置 |
| 虚拟机之间 | × | √ | 同网络名称可以 | √ |

Charset

计算机中都是二进制符号，编码后变成英文、中文、日文等。

  • 二进制转换成文字，过程叫解码。文字转二进制，叫编码。
  • 不同码表包含文字不同。比如ISO不包含中文。

主流编码

Ascii

最开始的编码，包括英文字母和键盘符号。

iso-ISO-8859-1

ascii无法表示欧洲字符，比如法语音标等。于是出现了iso-8859-1，其中包含了西欧字符。

GBK - 国标码

英文和中文，都是两个字节(16)。根据包含文字多少，分为

码表 描述
GBK-2312 全部汉字、非汉字符号
BIG-5 包括未被 GB 2312 收入的 139 个图形符号
nicode

为了包含全部语言，出现了unicode。

包括UTF-8、UTF-16、UTF-32

静态方法

availableCharsets - 所有编码
defaultCharset - 默认编码
forName - 获取编码

实例方法

name() / displayName() / alias() - 名称
encode() / decode() / newDecode() / newEncode() -编码解码

  • CharDecoder
  • CharEncoder

Reader/Writer

Stream读写文本不方便，于是有了Reader和Writer。如下

  • Reader和Writer会进行编解码，要设置编码。
  • Reader和Writer是对Stream的装饰模式。

Reader/Writer接口

  • Reader实现了Readeable，Writer实现了Appendable

具体实现

包装器 - InputStreamReader/OutputStreamWriter
  • 用于转换
  • 要设置编码方式
文件读写器 - FileReader / FileWriter

用于读写文件。

缓冲 - BufferedReader / BufferWriter

提供了缓冲区。提供了readLine()和writeLine()

PrintWriter

用于输出各种格式字符，可以通过str、File、Stream、Writer方便的创建。

Scanner

读取字符使用Scanner。

System.out

用于给控制台输出。

  • 类型是PrinterStream，方法用法和PrinterWriter一样。

Stream

Stream是流，用于在目的地(文件、网络、内存数组)中读取数据。所以

  • 创建时候，要指定目的地
  • 根据读取方式不同，分为字节流(Stream)和字符流(Reader/writer)
  • 流在读取写入(没有字符/等待设备)时候是阻塞(当前线程阻塞)的。

通用接口 - InPutStream/OutputStream

  • 所有流的通用接口

具体实现

文件读取 - FileInputStream / FileOutputStream

用于读写文件，需要指定路径或者File。

缓冲 - BuffteredInputStream / BufferOutputStream

BufferedStream提供了缓冲区。

各种格式数据(类型转换) - DataInputStream / DataOutputStream

DataStream额外实现了DateInput和DataOutput。

回推流(预览) - PushbackInputStream

PushBackInputStream可以预览或者会推一个字节。

标准输入 - System.in

用来从键盘(终端)读取数据。

  • 当键入回车，表示终端有了数据
