v-model

v-model用于双向绑定，通俗来说

  • 在input可以修改data

实现方式是

  • 使用:value='xx'，给input绑定data，用于预览
  • 使用@input监控键盘事件，在事件回调函数中更新data

根据你的<input type=''>的type不同，监听的事件和绑定的property可能不同。

  • text 和 textarea 元素使用 value property 和 input 事件；
  • checkbox 和 radio 使用 checked property 和 change 事件；
  • select 字段将 value 作为 prop 并将 change 作为事件。

v-show

v-show和v-if区别

v-show的实现方式是，把模板渲染到Dom，如果条件成立，显示。如果条件不成立，设置display:none属性。

v-if则是，如果不显示，就在Dom中剔除。

也就是说，v-show在第一次渲染时，会开销大。但是在切换时，开销小。

元素复用

复用

在上面的例子中，当我把login_type由username切换到phone，Vue算法是如下做的切换

  • 检查旧的v-if块和新的v-if块，筛选出相同的标签(key不同，不是相同的标签)

  • 把<label>中的username换成phone

而不是

  • 删除div
  • 添加新的div

这样可以复用元素，避免Dom开销。

不复用 - key

如果你确实希望不复用v-if块的内容，可以加上key属性，表示他们是不同的标签，就不会复用了。

用户管理

在linux中，有三种用户。

  • Root(0) - 可以编辑、查看所有文件，
  • 虚拟用户(1~999) - 没有家，不能登录。
  • 普通用户(1000+) -

对于目录，r表示可以使用ls查看，x表示可以使用cd进入、可以创建文件。

如果想让其他用户，可以修改、查看，权限是rx(5)。

对于文件，r表示查看，w表示写，x表示执行。

如果想让其他用户，可以修改、查看，不能执行，设置成rw(6)。

对于多级目录，在你查看里面的文件时候，需要所有级别的目录都可以r，否则，你是看不了的。

修改权限 - chmod

chmod 用于修改权限。

  • o表示其他用户，u表示用户，g表示用户组，a表示所有。
  • + 表示添加权限，-表示减少权限，=表示设置权限。
  • r表示读，w表示写，x表示执行。

默认权限 - umask

对于文件，使用666减去umask。

  • 666 - 022 = 644
  • 如果减去后，权限是奇数，会+1。比如523，会变成624。

对于目录，使用777减去umask。

  • 777 - 022 = 755
