文件属性信息

在linux中，使用ls -li查看文件属性信息。

如下

  • Part I - Inode信息
  • Part II - 文件类型和权限信息
  • Part III - 硬链接个数
  • Part IV - 属主信息 和属组信息
  • Part V - 文件大小
  • Part VI - 修改日期
  • Part VII - 文件名称

Inode和Block

在linux中，文件由两部分组成。分别是Inode和Block。

  • Inode - 保存了文件名称、文件权限、文件内容的地址
  • Block - 保存了文件内容

两个文件的Inode相同，互为硬连接。

无法给目录创建硬连接。

Block对于磁盘IO效率影响？

  • 读取每个Block的时间是相同的。
  • Block小，IO效率会低，磁盘浪费小。
  • Block大，IO效率高，磁盘浪费多。

文件类型

文件类型，用来区分文件类型。

常见分类如下

  • - - 普通文件

  • d - 目录

  • l - 软连接

  • c/b - 设备文件/字符文件

  • s - socket文件

file - 查看文件类型

使用file查看文件类型。

设备文件、字符文件

b表示设备文件，c表示字符文件。

常用的字符文件如下

  • /dev/sda - 磁盘文件。
  • /dev/urandom - 字符文件，用于产生随机数。
  • /dev/zero - 字符文件，输入黑洞，可以用输入黑洞来填充文件。
  • /dev/null - 字符文件，输出黑洞，可以把输出信息重定向到输出黑的。
ln - 创建连接

ln表示创建链接。

  • -s - 表示创建软连接
df - 查看Block、Inode、分区信息

df 查看Inode或Block的使用情况。

  • -h - 人类可读方式
  • -i - 查看Inode信息

时间信息

文件时间分为三种

  • atime - 访问时间，Access Time。
  • Ctime - 修改时间，Change Time，表示修改Inode信息，比如名称、权限。
  • mtime - 修改时间，Modify Time，修改Block内容。
stat - 查看文件时间

stat用于查看文件时间相关信息。

Maven

Maven是一个依赖管理和项目构建工具。功能如下

  • 可以进行依赖管理和自动更新
  • 约定了统一的项目结构
  • 提供了丰富的插件，可以用来构建、编译、测试、打包、生成报告等。

Maven生命周期和插件

生命周期、插件、目标(goal)

生命周期、插件、目标(goal)可以如下理解

  • 目标(goal) - 可以执行的一个功能点
  • 插件 - 多个goal(功能点)的集合
  • 生命周期 - 在执行构建命令时候，用来绑定插件
生命周期阶段

生命周期由如下阶段

阶段 作用 默认执行的插件
Clean 清除target目录 maven-clean-plugin
Validate 检查pom.xml是否正确
Compile build项目 maven-resource-plugin、maven-compiler-plugin
Test 单元测试 maven-surefire-plugin
Package 打包 maven-jar-plugin、maven-war-plugin
Verify 检查集成测试结果
Install 安装到本地仓库 maven-install-plugin
Site 生成项目信息 maven-project-info-reports-plugin
Deploy 发布到远程仓库 maven-deploy-plugin
构建项目三种方式
  • 执行生命周期 - 实际是执行默认插件
  • 执行具体插件(执行某个目标)
  • 插件的goal绑定到生命周期，然后执行生命周期

AOP - 面向切面编程

AOP概念

AOP是面向切面编程。是指把横切逻辑(事务管理、缓存、性能监视、权限控制、日志记录等)单独抽取出来，在编译期或者运行期，把横切逻辑织入到目标类，返回代理类的过程。

AOP中的概念

主要有如下

  • 连接点(Join Point) - 可以织入横切逻辑的位置集合
  • 切点(Pointcut) - 实际织入横切逻辑的位置
  • 通知(Advice) - 横切逻辑，有前置通知、后置通知、异常通知、最后通知、环绕通知。
  • 切面(Aspect) - 切点和通知的集合
  • 织入(Weaving) - 创建代理类的过程
  • 目标类(Target) - 目标类
  • 代理(proxy) - 返回的代理类

Spring中的AOP

在Spring中，实现aop有两种方式

  • spring-aop中自己实现了aop。
  • spring-aspectj整合了aspectweaver，提供了更加高效和方便的实现。

这里主要是spring-aop方式实现的aop。

AOP联盟定义了org.aopalliance.aop.Interface.Advice，Spring有如下Advice

  • MethodBeforeAdvice - 前置通知
  • AfterReturingAdvice - 后置通知
  • ThrowsAdvice - 错误通知
  • MethodInterceptor - 环绕通知

切点由MethodMatcher和ClassFilter组成，分别来拦截方法和类。

主要由如下ClassFilter

  • TrueClassFilter - 匹配所有

  • RootClassFilter - 匹配指定类

  • AnnotationClassFilter - 匹配标注了注解的类

  • UnionClassFilter - 组合ClassFilter

主要切点如下Pointcut

  • TruePointcut - 匹配所有
  • NameMatchMethodPointcut - 匹配方法
  • AnnotationMatchingPointcut - 匹配特定注解的方法
  • ExpressionPointcut - 正则切点
  • PerTargetInstantiationModelPointcut - 动态切点
  • ComposablePointcut - 组合切点

Advisor表示切面。

  • StaticMethodMatcherPointcutAdvisor - 相当于直接配置Advice
  • NameMatchMethodPointcutAdvusir
  • RegexpMethodPointcutAdvisor - 正则切面
  • DefaultPointcutAdvisor - 可以组合任意切点和Advice

自动注入详解

bean的autowrited属性

在使用autowrited属性的时候，有如下三个情况。

类型 解释
no/default 不自动装配。
byName 根据名称装配。找到就装配，找不到就是null。
byType 按类型装配。找到多个报错，找不到是null。
@autowrite/@qualified - 详细

使用@AutoWrite时候，默认按照类型装配。

  • 默认按type装配。找到多个Bean直接报错。找不到也报错。
  • 可以配合@Qualified按照名称装配。找到就装配，找不到就报错。
  • 可以使用require=false属性，在找不到时候，注入null。
@Resource - jsr250装配注解
  • @Resource是Jsr204提供的注入注解，对比@Autowried好处是，可以在非Spring中使用(代码迁移到别的框架，依旧有效)。
  • @Resource - 等同于 @Autowried，按照类型注入
  • @Resource(“name”) - 等同于@Qualified和@AutoWited，按照名称注入
