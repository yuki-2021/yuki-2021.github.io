宽表模式

简单来说，宽表模式是把一个对象的所有属性放到一个表中。

会有如下问题

  • 会有一些列出现冗余数据
  • 更新时候，对于冗余的行，需要更新多行。

解决方法

  • 把数据冗余的列，抽取出来

适用情况

  • 配合列存储的报表应用

数据库设计范式

对于数据库，有如下三个范式

范式一 - 字段不可以再分开

范式二 - 有业务主键，其他列依赖于业务主键

解释来说

  • 不能存在不依赖主键的列

在下面，用户积分不依赖于主键，所以不行。

范式三 - 非主键列之间不存在依赖关系

实际设计

在实际设计时候，不会完全遵守数据库设计三范式，会做一些调整。比如

  • 一般不会使用业务主键，而是使用逻辑主键
  • 对于满足三范式之后，但是还是会数据冗余的列，会抽取出``
  • 在关联表时候，不会单独抽取出关系表，仅仅对于多对多关系抽取。一对一或者一对多不会淡出抽取出表。

MySQL常见存储引擎

如下

事务 说明
MYISAM N 5.6默认，非事务引擎。 数据以堆表存储，没有聚集索引概念，叶子节点指向数据地址，避免了二次查找。读写加锁，频繁读写，大量阻塞。
CSV N CSV格式，不支持事务。读写加锁。用于不同数据库的数据交换。
Archive N 不能修改，只能查询和新增数据。用于日志和归档，物理大小小。
Memory N 内存，读写快。MYSQL内部临时存储数据集，加快SQL速度。
InnoDB Y 支持事务

InnoDB存储引擎特点

  • 支持ACID事务(事务引擎表和非事务表不能同时用，回滚时，非事务表不能回滚 )
  • 按照主键聚集存储，叶子节点指向主键而不是物理位置。
  • 支持行级别锁和MVCC
  • 支持Btree索引和Hash索引()
  • 支持全文索引和空间索引

数据类型特点

整型特点

  • zerofill仅仅用来做零填充，对于大小没有意义。

小数 - float、double、decimal

  • flocat不精确，运算后和实际会有偏差。但是测试发现，对于对于floca规定了长度，没有发现问题，在长度边长之后，还是有问题。

时间类型

  • 存储IP地址，使用整数。配合INTE_ATON、INET_NTOA

  • 对于备注，一般使用varchar。

  • 临时表不支持text,在排序时候不会创建出MEM 类型临时表。如果必须使用TEXT，建议分出一个表。Text只能是前缀索引。

  • 谨慎使用Text和 Enum，因为在修改时候，会加元数据锁

  • 财务数据，必须使用decimal。因为财务，会有大量小数和运算。

数据类型

Mysql数据类型分为整型、小数、字符型、日期型

整数类型

整数类型分为tinyint、smallint、mediumint、int、bigint

tinyint -2^7 ~ 2^7 -1
smallint -2^15 ~ 2^15 -1
midumint -2^23 ~ 2^23 -1
int -2^31 ~ 2^31 -1
bigint -2^63 ~ 2^63 -1
zerofill

可以配合zerofill进行0填充，在长度不够的时候，前面会补充0。

小数类型

小数类型有float(M,D)、double、decimal。

  • 浮点数，M和D默认没有限制。定点数，默认M=10,D=0。可以理解decimal = float(10,0)
  • 对于float和double指定M和D后，和decimal没有区别。
  • 插入数据，小数超过长度，自动截断。位数不够，自动补充0。
float -3.4E^38 - 3.4E^38
double -1.7E^308 - 1.7E^308
decimal 默认相当于double(10,0)

字符类型

字符类型包括char、varchar、tinyText、tinyBlob、blob、text、mediumtext、mediumBlob、longText、longBlob。

char和varChar

char是定长的，var是不定长度的。

  • 使用char(n)或者varChar(n)指定最多存储字符个数
  • char最大空间是255byte，varChar是65535。
char 最大225byte 定长，效率高
varchar 最大65535byte 不定长度，效率低
其他文本

存储大文本用tintText、text、mediumText、longText，差别只是存储内容多少区别

tinaytext 2^8byte,对应85个utf8字符
text 2^16byte
mediumtext 2^24byte
longtext 2^32byte

分别和tinyblob、blob、mediumblob、longblob对应，用来存储二进制。

set和enum

enum表示枚举。

  • 创建表时，声明值的范围
  • 插入时候，只能插入声明值的一个
  • 查询时候，可以使用enum对应的int值查询。
  • 约束值本来是check的功能，但是mysql不支持。有需求时，可以使用enum或者trigger。

在查询的时候，可以使用enum对应的整数查询

set类型

set类型表示集合类型

  • 创建表，声明值的范围
  • 插入数据，只能插入声明值的一个或多个
  • 查询时候，使用内置函数查询

日期类型

包括year、date、time、timedate、TIMESTAMP。

类型 范围 插入 查询
year 1901-2155 插入超出范围，报错 查询出来
date 0-00-00 到 9999-12-12。错误时间的时间戳是0。 分隔符做符号,比如y@m@d。插入值是str或者date。 查询结果自动格式化成str
time
datetime 0-00-00 到 9999-12-12。
错误时间的时间戳是0。		 分隔符做符号,比如y@m@d。
支持插入值是str或者date。		 查询结果自动格式化成str
timestamp 最大2147483647 默认是当前时间。
支持插入str或者date。		 查询结果自动格式化成str

约束

  • 包括null、default、unique、primaryKey、foreignKey，不支持check。
  • primaryKey和foreignKey一般显示指定constraint名称
null 值不能是null
default 默认值
unisque 值唯一
primaryKey 主键
foreignKey 外键
